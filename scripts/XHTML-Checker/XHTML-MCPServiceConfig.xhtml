MCP Server Overview

The MCP Service Config tool provides a centralized GUI for managing all Model Context Protocol servers defined in .vscode/mcp.json. Access it from Tools → MCP Service Config in the main GUI.

Server Type Runtime Purpose
tplink-omada-mcp stdio Docker TP-Link Omada SDN controller management (APs, switches, gateways, clients)
fing-mcp stdio Node Fing network scanner local API (device discovery, bandwidth monitoring)
mikrotik-mcp stdio Docker MikroTik RouterOS SSH management (firewall, routing, interfaces)
lifx-api-mcp stdio npx LIFX smart lighting control (scenes, effects, scheduling)
flipper-mcp http — Flipper Zero RF toolkit (Sub-GHz, NFC, RFID, IR, GPIO)
tapo-mcp http — TP-Link Tapo smart home devices (plugs, cameras, sensors)
tuya-mcp stdio Python Tuya IoT platform integration (multi-device cloud control)

Server Reference

tplink-omada-mcp Docker

Connects to the TP-Link Omada SDN controller via OAuth. Requires Omada controller URL, client ID, client secret, and omadacId.

{
  "type": "stdio",
  "command": "docker",
  "args": ["run", "--rm", "-i",
    "-e", "OMADA_BASE_URL=...",
    "-e", "OMADA_CLIENT_ID=...",
    "-e", "OMADA_CLIENT_SECRET=...",
    "-e", "OMADA_OMADAC_ID=...",
    "-e", "OMADA_STRICT_SSL=false",
    "jmtvms/tplink-omada-mcp:latest"]
}

fing-mcp Node

Local Fing agent API integration. Requires the Fing API key and base URL (default: http://localhost:49090/1).

{
  "type": "stdio",
  "command": "node",
  "args": ["${workspaceFolder}/../fing-mcp/dist/index.js"],
  "env": { "FING_API_KEY": "...", "FING_BASE_URL": "..." }
}

mikrotik-mcp Docker

SSH-based router management for MikroTik devices. Requires host IP, username, and password.

{
  "type": "stdio",
  "command": "docker",
  "args": ["run", "--rm", "-i",
    "-e", "MIKROTIK_HOST=...",
    "-e", "MIKROTIK_USERNAME=...",
    "-e", "MIKROTIK_PASSWORD=...",
    "-e", "MIKROTIK_PORT=22",
    "mikrotik-mcp"]
}

lifx-api-mcp npx

Cloud-based LIFX smart lighting. Requires a personal access token from cloud.lifx.com.

{
  "type": "stdio",
  "command": "npx",
  "args": ["-y", "lifx-api-mcp-server@latest"],
  "env": { "CONFIG_API_TOKEN": "..." }
}

flipper-mcp http

Flipper Zero RF multi-tool over HTTP. The device must be running the MCP firmware module and reachable on the local network.

{
  "type": "http",
  "url": "http://flipper-mcp.local:8080/mcp"
}

tapo-mcp http

TP-Link Tapo smart home bridge. The local tapo-mcp service must be running on port 3000.

{
  "type": "http",
  "url": "http://127.0.0.1:3000/mcp",
  "headers": { "Content-Type": "application/json" }
}

tuya-mcp Python

Tuya IoT cloud integration via the official MCP SDK. Requires Access ID, Access Secret, and endpoint from the Tuya developer platform.

{
  "type": "stdio",
  "command": "python",
  "args": ["-m", "examples", "all",
    "--endpoint", "...",
    "--access-id", "...",
    "--access-secret", "..."],
  "cwd": "${workspaceFolder}/../tuya-mcp-sdk/mcp-python"
}

Configuration Guide

File Location

All MCP server definitions live in .vscode/mcp.json at the workspace root. This file contains two top-level sections:

  • inputs — Credential prompts (API keys, passwords, URLs) used via ${input:id} references
  • servers — Named server definitions with type, command, args, env, and cwd

Adding a New Server

Use the + Add Server tab in the MCP Service Config tool, or manually edit mcp.json:

  1. Open .vscode/mcp.json
  2. Add input prompts under "inputs" for any credentials
  3. Add the server definition under "servers"
  4. Reference inputs with ${input:your-input-id}
  5. Use Backup Config before making changes

Server Types

stdio — VS Code spawns the process and communicates over stdin/stdout. Define command, args, and optionally env and cwd.

http — VS Code connects to an already-running HTTP endpoint. Define url and optionally headers.

Credential Security

Sensitive inputs use "password": true so VS Code masks them in the prompt. Credentials are never stored in mcp.json — they are prompted at runtime via the inputs mechanism. For vault-backed secrets, see the Secrets Management Guide.

Backups

The MCP Service Config tool stores timestamped backups in config/mcp-backups/. A backup is automatically created when adding a new server through the GUI. Use Backup Config on the main tab before any manual edits.

Operations Guide

Testing Servers

stdio servers: The tool checks whether the command (docker, node, npx, python) is available in PATH. This does not start the server — it verifies the runtime is installed.

http servers: The tool sends an HTTP HEAD request to the configured URL with a 3-second timeout. A successful response means the server is reachable.

Status Indicators

StatusColourMeaning
ReachableGreenHTTP endpoint responded successfully
InstalledGreenstdio command found in PATH
MissingRedstdio command not found
UnreachableRedHTTP endpoint did not respond

Editing Server Config

Each server has its own tab with a JSON editor. Edit the definition, then click Save to write changes to mcp.json. Click Reload to discard changes and reload from disk.

Docker-Based Servers

For servers using Docker (tplink-omada-mcp, mikrotik-mcp), ensure Docker Desktop is running. The containers are configured with --rm so they auto-clean after stopping. Pull the latest image with docker pull <image> before first use.
