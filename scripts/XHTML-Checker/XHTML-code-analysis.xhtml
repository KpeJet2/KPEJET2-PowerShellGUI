1. Functional Purpose — Sequential Nested Hierarchy

The application is a Windows Forms-based PowerShell GUI that provides IT administrators with a unified launcher for maintenance scripts, configuration management, version control, and system diagnostics. Below is the execution flow presented as a nested hierarchy.

  • Launch-GUI.bat — Top-level entry point; presents numbered menu
    • Launch-GUI-quik_jnr.bat — Fast startup (detects pwsh 7+ or falls back)
      • Calls Main-GUI.ps1 -StartupMode quik_jnr
    • Launch-GUI-slow_snr.bat — Full checks startup
      • Calls Main-GUI.ps1 -StartupMode slow_snr
    • Launch-GUI-BUILD-VER.bat — Build/version mode
      • Calls Main-GUI.ps1 -StartupMode quik_jnr
    • Dynamic scan of .bat, .ps1, .html, .xhtml files
  • Main-GUI.ps1 — Core application (~4 400 lines)
    • Phase 0: Initialisation & Assembly Loading
      • Load System.Windows.Forms, System.Drawing, System.Web
      • Initialize XML cache ($script:_XmlCache) and log buffer ($script:_LogBuffer)
      • Define local path variables: $ConfigPath, $DefaultFolder, $TempFolder, $ReportFolder, $DownloadFolder
      • Define global path variables: $configDir, $modulesDir, $logsDir, $scriptsDir
      • Create directories if missing
      • Request-LocalPath() — single-field prompt with timeout
      • Request-LocalPathsUnified() — multi-field prompt with timeout
      • Show-ConfigMaintenanceForm() — folder management GUI (nest/move/copy/archive/clear)
    • Logging Functions
      • Write-AppLog() — buffered application log writer
      • Write-ScriptLog() — per-script log writer
      • Flush-LogBuffer() — flush grouped log entries to disk
    • Configuration Functions
      • Initialize-ConfigFile() — create system-variables.xml with defaults
      • Get-ConfigVariable() — read single XML node value
      • Get-ConfigSubValue() — cached XPath-based config read
      • Set-ConfigSubValue() — XPath-based config write with auto-create
      • Get-ConfigList() — return folder exclusion list from XML
      • Save-ConfigPathValues() — persist path hashtable to XML
      • Get-ButtonConfiguration() — load left/right button arrays from XML
      • Initialize-LinksConfigFile() — create links.xml with defaults
      • Get-LinksConfig() — cached links.xml reader
      • Get-VersionInfo() — consolidated Major.Minor.Build retrieval
      • Build-LinksMenu() — populate ToolStripMenuItem from links.xml
    • Version & Tagging Functions
      • Update-VersionBuild() — increment build number in XML
      • Update-VersionTag() — stamp version comment tags across workspace
      • Test-VersionTag() — check all files for tag consistency, output diff XML
      • Compare-ExcludedFolder() — manifest cross-reference for excluded folders
      • New-BuildManifest() — generate build manifest with SHA256, owner, cert info
    • Packaging & Security Functions
      • Export-WorkspacePackage() — zip workspace with optional encryption
      • Show-BuildPackageOptionsForm() — options GUI for packaging
      • Clear-FolderContents(), Remove-ItemsOlderThan()
      • Resolve-7ZipPath(), New-EncryptedZip()
      • Get-OrCreate-CodeSigningCert() — self-signed cert management
      • Get-ScriptSafetyScore() — heuristic safety scoring (0–100)
    • Script Execution Functions
      • Invoke-ScriptWithElevation() — launch script with optional RunAs
      • Invoke-LocalScriptWithProgress() — job-based execution with rainbow progress
      • Wait-ProcessWithProgress() — elevated process progress monitor
      • Show-ElevationPrompt() — safety-badge admin elevation dialog
      • Get-RainbowColor(), Write-RainbowProgress()
    • Diagnostics & Testing Functions
      • Get-NetworkDiagnostic() — IP, gateway, WAN, DNS checks
      • Show-DiskCheckDialog() — chkdsk launcher
      • Show-SystemCheck() — sfc /detectonly wrapper
      • Show-PrivacyCheck() — ms-settings:privacy launcher
      • Show-WingetInstalledApp() — registry-based app inventory
      • Show-WingetUpgradeCheck(), Show-WingetUpdateAllDialog()
      • Test-AppTesting() — markdown/comment/regression tests
      • Test-ScriptSafetySecOp() — safety scoring for all scripts
    • Path Management Functions
      • Test-PathReadWrite() — R/W access probing
      • Show-PathSettingsGUI() — path configuration dialog
      • Get-ScriptFoldersConfig(), Save-ScriptFoldersConfig()
      • Get-AllScriptFolders(), Show-ScriptFolderSettingsGUI()
    • Phase 1 (slow_snr only): Version Tag Consistency
      • Calls Test-VersionTag()
      • Parses diff XML for mismatches
      • Prompts user for auto-increment (“AA”)
    • Phase 2: Auto-Increment (conditional)
      • Calls Update-VersionBuild() then Update-VersionTag()
    • Phase 3 (slow_snr only): Build Manifest Generation
      • Calls New-BuildManifest()
    • Phase 4: Display System Info & Launch GUI
      • New-GUI() — construct and show main Windows Form
        • Menu Bar: File, Tests, Links, WinGets, Tools, Help
        • Left Column: 6 configurable script-launch buttons
        • Right Column: configurable script-launch buttons
        • Status Bar: computer, user, paths
    • Shutdown
      • Calls Flush-LogBuffer()
  • modules/
    • AVPN-Tracker.psm1 — VPN connection tracker module
    • PwSh-HelpFilesUpdateSource-ReR.psm1 — Help update source
    • PwShGUI_AutoIssueFinder.psm1 — ScriptAnalyzer-based issue finder with XHTML reporting
  • scripts/ — Script1–6, Script-A–F, QUICK-APP
  • config/ — system-variables.xml, links.xml, JSON configs
  • View-Config.ps1 — Console-based config and log viewer
  • fix_check_version.ps1 — Patch: rewrites Check-VersionTags in Main-GUI.ps1
  • fix_update_version.ps1 — Patch: rewrites Update-VersionTags in Main-GUI.ps1

2.3 Error Handling Gaps (potential for halted execution)

IssueLocationRecommendation
$ErrorActionPreference = "Stop" at script scope Main-GUI.ps1:50 Any uncaught exception terminates the entire script. Wrap Phase 0–4 in try/catch blocks or scope $ErrorActionPreference to individual functions.
$script:_LogBuffer.Clear() on non-ArrayList Main-GUI.ps1:1255 $script:_LogBuffer is initialized as @() (array). .Clear() method exists on [System.Collections.Generic.List] but not on a PowerShell array. Use $script:_LogBuffer = @() instead.
Get-ConfigVariable null dereference Main-GUI.ps1:1396 $xml.SystemVariables.$VariableName returns $null if node does not exist, then .InnerText fails. Add null check.
Read-Host blocks GUI in slow_snr mode Main-GUI.ps1:4298 The version-tag mismatch prompt uses Read-Host from the console, which blocks if launched from a non-console host. Consider a Forms dialog fallback.
AVPN module import in $ErrorActionPreference = "Stop" Main-GUI.ps1:1259–1263 If Import-Module fails for any reason other than file-not-found (e.g., missing dependency), it throws a terminating error. Wrap in try/catch.
Test-VersionTag calls Get-ConfigList inside Where-Object Main-GUI.ps1:2441 Get-ConfigList "Do-Not-VersionTag-FoldersFiles" is called once per file. Move outside the pipeline for performance and to avoid repeated XML reads.

2.4 Normalisation & Optimisation Opportunities

CategoryCurrent StateProposed Normalisation
Path Variables Local paths hardcoded at lines 1116–1120; global paths derived from $scriptDir at lines 1145–1151. Both sets serve similar purposes. Consolidate into a single $AppPaths hashtable or indexed array. Example: $AppPaths = @{ Config = ...; Default = ...; Temp = ...; Report = ...; Download = ... }
Remote Path Variables Seven individual $Remote*Path variables (lines 1155–1161) and parallel definitions in $remotePathDefs array (lines 776–784). Use a single $RemotePaths ordered hashtable: $RemotePaths = [ordered]@{ Update = ''; Config = ''; Template = ''; Backup = ''; Archive = ''; Links = ''; Download = '' }
Excluded Folders Exclusion list defined in XML Do-Not-VersionTag-FoldersFiles and re-read via Get-ConfigList in 5+ locations. Read once at startup into $script:ExcludedFolders array and pass as parameter.
GUI Dimensions Button width (260), height (50), spacing (10), column positions (30, 310) are hardcoded in New-GUI. Define as a $GUILayout hashtable: @{ BtnWidth=260; BtnHeight=50; Spacing=10; Col1X=30; Col2X=310; StartY=90 }
Log Level Strings "Info", "Warning", "Error", "Success", "Debug", "Event" are string literals throughout. Define as an enum or constant array: $LogLevels = @('Info','Warning','Error','Success','Debug','Event')
Font Definitions New-Object System.Drawing.Font("Arial", 10) repeated 10+ times across GUI functions. Pre-create font objects: $Fonts = @{ Normal = [System.Drawing.Font]::new("Arial",10); Bold = ... }
Safety Pattern Arrays $patterns (10 items) and $secretPatterns (5 items) in Get-ScriptSafetyScore are rebuilt on every call. Move to script-scope $script:SafetyPatterns and $script:SecretPatterns initialized once.
Duplicate Code in Grid Selection The pattern “iterate $dgv.Rows, check Select checkbox, build $selected array” is repeated 6 times in Show-ConfigMaintenanceForm. Extract helper: function Get-SelectedRows($dgv, [string]$filter) { ... }

3. Proposed Testing Routine

The automated test script tests/Invoke-AutomatedTests.ps1 implements the following ten test categories. Each test emits structured results with Status (PASS/FAIL/WARN/INFO/ERROR), Test name, File, and Detail.

#Test CategoryWhat It Checks
1Syntax Parse ValidationParses every .ps1/.psm1 file using [Parser]::ParseInput(). Reports syntax errors.
2Version Tag ComplianceChecks each file for VersionTag presence, duplicate tags, and value match against config XML.
3Function Naming & SignaturesValidates Verb-Noun naming against Get-Verb approved verbs list.
4Error Handling CoverageScans each function body for try/catch or -ErrorAction patterns.
5Variable ConsistencyDetects automatic variable shadowing and hardcoded filesystem paths.
6XML Well-FormednessParses all .xml config files to confirm they are valid XML.
7JSON Well-FormednessParses all .json config files via ConvertFrom-Json.
8Markdown StandardsChecks for H1 headings, trailing whitespace, TODO/TBD markers.
9Batch File ValidationChecks for @echo off, setlocal, and endlocal directives.
10Dotfile StandardsValidates .gitignore for expected entries, duplicate tags, and trailing blanks.

Usage

# Run all tests against the full repository
.\tests\Invoke-AutomatedTests.ps1

# Run tests against a specific script
.\tests\Invoke-AutomatedTests.ps1 -TargetPath .\Main-GUI.ps1

# Output to log file only
.\tests\Invoke-AutomatedTests.ps1 -OutputFormat Log
