# FocalPoint-null

**Version**: 1.0.0  
**Python**: 3.10+  
**Framework**: [Microsoft Agent Framework](https://github.com/microsoft/agent-framework) (pinned preview SDK)  
**Default Model**: `gpt-4.1` (GitHub Models) — swap to Foundry via `.env`

---

## Overview

FocalPoint-null is a production-grade multi-agent orchestration system. It is designed around a **root orchestrator** (`FocalPoint-null-00`) that brokers all task allocation, agent communication, human review, security proxying, and lifecycle management.

### Core Philosophy

- **FocalPoint-null-00 never writes, reviews, or tests code directly.** Code operations are delegated to sub-agents that have completed ≥1 full orchestration loop.
- **All subagent outputs pass through `ViewPoint-init`** — a security proxy that encodes results as base64 transforms before FocalPoint processes them. This shields against direct memory, side-channel, OOB and speculative execution attacks.
- **All inter-agent messages are brokered by FocalPoint-null-00** and signed with PKI (RSA-2048).
- **Forks iterate until complete and tested, then lock** — preventing exponential autonomous growth.

---

## Agent Roster

| Agent ID | Role | Capabilities |
|---|---|---|
| `FocalPoint-null-00` | Root orchestrator | Task allocation, role control, brokering, checkpoints, human review |
| `ViewPoint-init` | Security proxy | Encodes all subagent output before FocalPoint sees it |
| `SecurityTriage-00` | Security | Threat detection + sandboxed remediation (client-sim validated) |
| `AgentReview-00` | Review | Performance review, regression detection, role suggestions |
| `QualityControl-00` | Quality | Code quality, refactoring, standards uplift (min 1 version loop required) |
| `StandardsGuard-00` | Standards | Schema updates, template management, admin TO-DO |

**Sub-orchestrators**: If a second orchestrator is ever recommended, it is created as `FocalPoint-null-01`, incrementing for each additional one.

## Quick Start

### 1. Prerequisites

- Python 3.10+
- A GitHub Personal Access Token (free) **OR** a Microsoft Foundry deployment

### 2. Set Up Environment

```powershell
# From the focalpoint-null/ directory:
cd c:\PowerShellGUI\agents\focalpoint-null

# Create virtual environment
python -m venv .venv
.venv\Scripts\Activate.ps1

# Install dependencies (--pre required for agent-dev-cli)
pip install -r requirements.txt --pre
```

### 3. Configure

```powershell
# Create your .env from the template
Copy-Item .env.example .env

# Edit .env and set GITHUB_TOKEN (or FOUNDRY_PROJECT_ENDPOINT)
# IMPORTANT: Change PKI_MASTER_SECRET to a strong random value
```

### 4. Run

**HTTP Server mode (recommended — works with AI Toolkit Agent Inspector):**
```powershell
python main.py --server
# Or use F5 in VS Code with "Debug FocalPoint-null HTTP Server"
```

When server mode starts, it also starts a monitoring dashboard on port `8888` by default.
- Dashboard: `http://127.0.0.1:8888/`
- Help page: `http://127.0.0.1:8888/help`
- Live monitor API: `http://127.0.0.1:8888/api/monitor`

**CLI mode (quick test):**
```powershell
python main.py --cli --input "Review the current agent setup and suggest improvements"
```

### Monitoring Dashboard

The dashboard shows live per-agent telemetry for:
- status (`idle`, `active`, `error`)
- URL/port link
- last message received time
- last message sent time
- error count and last error message
- uptime duration
- active processing duration

### Dashboard Configuration (`.env`)

| Setting | Default | Description |
|---|---|---|
| `DASHBOARD_HOST` | `127.0.0.1` | Dashboard bind host |
| `DASHBOARD_PORT` | `8888` | Dashboard bind port |
| `DASHBOARD_AUTO_OPEN` | `true` | Auto-open dashboard in browser at startup |

## Security Model

### ViewPoint-init Protection
All subagent outputs pass through `ViewPoint-init` before FocalPoint-null processes them:
1. Sanitizes dangerous patterns (exec, eval, shell injection, script tags, null bytes)
2. Base64 encodes the sanitized output
3. Computes SHA256 integrity hash
4. Returns `ViewPointResponse` with restricted recipient list
5. FocalPoint calls `response.decode_for(focalpoint_id)` — access denied if not in permitted list

### PKI Inter-Agent Authentication
- Each agent has an RSA-2048 keypair generated at bootstrap (`pki/<agent_id>.pem`, `.pub`, `.key`)
- All brokered messages are signed by FocalPoint-null-00's private key
- Verified by recipient using the sender's registered public key
- Certificate fingerprints (SHA256) stored in `config/agent_registry.json`

### Code Operations Guard
- `FocalPoint-null-*` agents: **NEVER** write, review, or test code
- Code-capable agents (`QualityControl-00`): require `version_loops_completed >= 1`
- All code operations must complete at minimum one full orchestration loop before being permitted

### SecurityTriage Isolation
- All SecurityTriage results pass through ViewPoint-init before returning to FocalPoint
- Client simulation testing is required before remediation is considered complete
- SecurityTriage outputs are never directly executed

## Checkpoint & Resume

FocalPoint-null saves a signed checkpoint after every orchestration loop. On startup, it automatically resumes from the latest valid checkpoint.

**Manual resume from specific checkpoint:**
```python
checkpoint_mgr.load_checkpoint_by_id("<checkpoint_id>")
```

**Apply replay log after disconnection:**
```python
state = checkpoint_mgr.apply_replay_log("path/to/replay.json", current_state)
```

## Admin TO-DO

The `todo/` directory contains categorised TO-DO items for admin review. These are **completely separate from the code project** and contain recommendations from FocalPoint-null and sub-agents.

Categories:
- `new_agents` — Suggested new agent roles
- `agent_changes` — Recommended changes to existing agents
- `human_oversight_required` — Items requiring human approval
- `standards_updates` — Standards document improvements
- `security_recommendations` — Security improvements

**Review `todo/ADMIN-TODO.json` before first production run.**

## Access Logs & Security Monitoring

FocalPoint-null maintains separate access counters for:
- **FocalPoint-null-00** (file, website, memory accesses, prompts, checkpoints)
- **Each sub-agent** (same categories, tracked independently)

Summary available via:
```python
log_manager.get_access_summary()
# Returns: secured/unsecured call counts, call methods, process/thread IDs
```

All logs are written to `logs/` as structured JSONL. Archives are SHA256-signed with optional `ALLOW-AGENT-ARCHIVAL-ACCESS` metatag for PKI-gated agent re-access.
