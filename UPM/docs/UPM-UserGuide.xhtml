User Profile Manager

Complete visual guide — v1.4.0 · PowerShell 5.1+ · Windows 10/11

Overview

User Profile Manager (UPM) is a Windows PowerShell WinForms application that captures, saves, compares, and restores Windows user profile settings across 23 categories — from installed applications and PowerShell environments to display layout, certificates, fonts, and regional settings.

Profiles are stored as .upjson files and can optionally be protected with AES-256-CBC + PBKDF2-SHA256 encryption. A full diff engine lets you compare any saved snapshot to the current live state, and the RESTORE tab lets you selectively replay changes with an automatic rollback snapshot.

Requirements

  • Windows 10 or Windows 11
  • PowerShell 5.1 or PowerShell 7+
  • winget — Windows Package Manager (for Winget Applications capture)
  • Windows Terminal (optional — for Terminal Configuration capture)
  • PowerShell ISE (optional — for ISE Configuration capture)
  • Run with Administrator rights to capture machine-scope items (drivers, fonts, HKLM)
⚠ UAC Elevation Some capture categories (Print Drivers, Machine Fonts, HKLM registry) require elevation. The status bar shows ▲ ADMIN when running elevated. Launch via Launch-GUI.bat which prompts for elevation automatically.

Folder Structure

Launching

Double-click Launch-GUI.bat in the parent PowerShellGUI\ folder, or run directly:

# From PowerShell (auto-elevates if not admin)
Set-Location 'c:\PowerShellGUI\UPM'
.\UserProfile-Manager.ps1

The launcher batch file calls PowerShell.exe -ExecutionPolicy Bypass so no profile-wide execution policy change is needed.

Status Bar

The status bar at the bottom of the window is split into four segments:

alice@contoso.com
DESKTOP-XPS15
▲ ADMIN
Ready — 0 profiles in store

  • UPN — Active Directory User Principal Name (user@domain), or USERNAME@USERDNSDOMAIN for local accounts
  • Computer — NetBIOS computer name ($env:COMPUTERNAME)
  • Admin — ▲ ADMIN (amber) when elevated, ● USER otherwise
  • Status — live operation feedback (capture progress, save confirmation, errors)

Menu Bar

  • New Capture — triggers a full live capture (same as the CAPTURE button on VIEW tab)
  • Save Profile — opens the SAVE tab

  • Open Log Folder — opens %TEMP%\UPM-Backups in Explorer

  • User Guide — opens this XHTML file in your default browser
  • Version History — opens VERSION-HISTORY.md in VS Code / Notepad

VIEW Tab

Shows a live tree view of the current user profile state. Click CAPTURE NOW to run all 23 capture functions and populate the tree. Expand each category node to inspect sub-items. The capture runs asynchronously and updates the status bar with progress.

💡 Tip Run a capture first before switching to COMPARE or RESTORE — those tabs require a live snapshot.

SAVE Tab

Saves the current live snapshot to a .upjson profile file.

  • Enter a Profile Name (used as the file base name and log sub-directory)
  • Choose a Save Path (defaults to the configured Profile Store folder)
  • Toggle Encrypt to protect the profile with AES-256-CBC encryption — you will be prompted to enter and confirm a password
  • Click SAVE PROFILE

COMPARE Tab

Diff the current live snapshot against any saved profile.

  • Select a saved profile from the Profile combo box
  • Click COMPARE — the grids populate with colour-coded differences
  • Green rows = items present in the saved profile but absent now (removed)
  • Amber rows = items present now but absent in the saved profile (added)
  • Each category has its own sub-tab within the Compare view

RESTORE Tab

Selectively restore settings from a saved profile back to the live system.

  • Select a source profile
  • Tick the categories you want to restore
  • Enable Create Rollback Snapshot (recommended) — saves the current state first
  • Click START RESTORE
  • Review the log panel — ✔ Restored, – Skipped, ✘ Error
  • Use ROLLBACK to revert if something went wrong

Save Code Examples

Each capture function can be called independently from the PowerShell prompt after importing the module.

# Import the module
Import-Module 'c:\PowerShellGUI\UPM\modules\UserProfileManager.psm1'

# 1 — Winget Applications
$apps = Get-WingetApplications
$apps | Select-Object Name, Id, Version | Format-Table

# 2 — PowerShell Environment
$psenv = Get-PSEnvironment
$psenv.Modules | Select-Object Name, Version | Sort-Object Name

# 7 — WiFi Profiles
$wifi = Get-WiFiProfiles
$wifi | Select-Object ProfileName, Authentication, Encryption

# 9 — Certificates
$certs = Get-CertificateStores
$certs.UserStore | Select-Object Subject, Thumbprint, NotAfter

Restore Code Examples

Categories marked Yes for Restore have automated restore logic inside the module. Categories marked No can be restored manually using the patterns below.

# 1 — Restore Winget Applications (automated)
# The RESTORE tab runs the equivalent of:
foreach ($app in $profile.Data.WingetApplications.Missing) {
    winget install --exact --id $app.Id --silent --accept-source-agreements
}

# 3 — User App Config registry (manual pattern)
foreach ($key in $profile.Data.UserAppConfigs.RegistryKeys) {
    foreach ($v in $key.Values) {
        Set-ItemProperty -Path $key.KeyPath -Name $v.Name -Value $v.Value
    }
}

# 12 — Register missing PS Repositories
foreach ($repo in $profile.Data.PSHelpRepositories.Missing) {
    Register-PSRepository -Name $repo.Name `
        -SourceLocation $repo.SourceLocation `
        -InstallationPolicy $repo.InstallationPolicy
}

# 18 — Reconnect Mapped Drives
foreach ($drive in $profile.Data.MappedDrives) {
    New-SmbMapping -LocalPath $drive.Drive -RemotePath $drive.NetworkPath -Persistent $true
}

WhatIf Simulation

The WhatIf Simulation dialog (Help → WhatIf Simulation) lets you preview what any save or restore operation would do without making any changes.

  1. Open Help → WhatIf Simulation
  2. Select one or more operations from the list (all 23 are shown)
  3. Choose Save mode or Restore mode
  4. Click Run WhatIf
  5. Review the HTML output in the results panel — it shows exactly which registry keys, files, or commands would be touched
💡 Tip WhatIf mode never writes to disk or registry. It is safe to run on production machines before committing to a full capture or restore.

Encryption

When you tick Encrypt on the SAVE tab, the profile data is protected with:

  • AES-256-CBC symmetric encryption
  • PBKDF2-SHA256 key derivation with 200,000 iterations
  • 32-byte random salt stored in the profile Meta envelope
  • 16-byte random IV generated per save

The password is never stored. To load or restore an encrypted profile you must provide the same password used when saving.

⚠ Password Loss There is no password recovery mechanism. If you lose the password the profile data cannot be decrypted.

Session Logging

UPM writes a single log file per session. The log path is:

%TEMP%\UPM-Backups\<ProfileName>\UserProfile-Manager-<COMPUTER>-<USER>-<yyyyMMdd-HHmmss>.log

The timestamp is fixed at startup so all log entries for a session go into one file. When you save a profile, the log directory is updated to use the profile name as the sub-folder via Update-LogProfile. Log entries look like:

[2026-03-01 14:23:18] [INFO]  Capture initiated by user
[2026-03-01 14:23:22] [INFO]  Capture complete: 847 apps, 12 modules, 23 categories
[2026-03-01 14:23:45] [INFO]  Save initiated: profile='Work-2026-03' path='D:\Profiles'
[2026-03-01 14:23:45] [INFO]  Save complete: D:\Profiles\Work-2026-03.upjson

[2026-03-01 14:30:55] [INFO]  Restore started: source profile='Work-2026-01'
[2026-03-01 14:31:02] [INFO]  Restore complete

Profile File Format

Profile files use the .upjson extension and contain a JSON object with two top-level keys:

{
  "Meta": {
    "ProfileName": "Work-2026-03",
    "CaptureDate": "2026-03-01T14:23:45",
    "Computer":    "DESKTOP-XPS15",
    "User":        "alice",
    "UPMVersion":  "1.4.0",
    "Encrypted":   false,
    "Salt":        null
  },
  "Data": {
    "WingetApplications": [ ... ],
    "PSEnvironment": { ... },
    /* ... 21 more data keys ... */
  }
}

For encrypted profiles, Data is a Base64-encoded string (the AES ciphertext) and Meta.Encrypted is true. The Salt field holds the hex-encoded 32-byte PBKDF2 salt.
