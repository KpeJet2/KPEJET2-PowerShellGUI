Vault Health Checklist

  • BW CLI installed (bw --version)
  • Server configured (cloud or self-hosted)
  • Login authenticated (bw login)
  • Vault unlocked (bw unlock)
  • Session key stored securely (env var or DPAPI)
  • Vault synced recently (bw sync)
  • Master password rotated within 90 days
  • 2FA/TOTP enabled on Bitwarden account
  • Emergency access configured
  • Vault export backup taken (encrypted JSON)
  • Weak/reused passwords identified and rotated
  • Windows Hello / biometric unlock enabled

Quick Reference & README

Installation

winget install Bitwarden.CLI or npm install -g @bitwarden/cli

Verify: bw --version (requires 2024.x+)

Authentication Flow

  1. bw config server https://vault.bitwarden.com (or self-hosted URL)
  2. bw login user@example.com (interactive master password prompt)
  3. bw unlock → outputs $env:BW_SESSION = "..."
  4. Set the session key: $env:BW_SESSION = "your-session-key"

Common Operations

bw list items --session $KEY List all vault items as JSON
bw get item "Site Name" --session $KEY Get a specific item by name or UUID
bw generate -ulns --length 24 Generate a 24-char password (upper, lower, number, special)
bw export --format encrypted_json --session $KEY Export encrypted vault backup
bw sync --session $KEY Force-sync vault with server

AssistedSASC Integration

This project uses AssistedSASC.psm1 for vault integration with Windows Hello (DPAPI). Session keys are encrypted at rest via Enable-WindowsHello and decrypted on-the-fly via Get-WindowsHelloPassword. See config/sasc-vault-config.json.

Security Notes

  • Never store the master password or session key in scripts (SIN P001)
  • Use $env:BW_SESSION in the current process only — do not persist to profile
  • Lock the vault when not in active use: bw lock
  • Encrypted exports use Argon2id key derivation — store the export password separately
